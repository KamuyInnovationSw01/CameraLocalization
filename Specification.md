# カメラローカライゼーション - ARマーカーポーズ推定システム 仕様書

**バージョン**: 1.0  
**日付**: 2026-07-01  
**ステータス**: 完成

---

## 1. 概要

このシステムは、USBカメラで撮影したARマーカー（ArUco）映像をリアルタイムで処理し、カメラの3次元ポーズ（位置と姿勢）を推定します。推定されたポーズは、ワイヤフレームの3D表示として2つのウィンドウに表示されます。

**主要目標**:
- リアルタイムでARマーカーを検出
- カメラのポーズを正確に推定
- 3D ワイヤフレーム（フラスタム・ボディ・光軸）で可視化
- シンプルで拡張性の高い設計

## 3. 機能要件

### 3.1 マーカー検出機能

| 要件 | 詳細 |
|------|------|
| **検出アルゴリズム** | OpenCV ArUco 検出器 |
| **対応辞書** | DICT_4X4_50（カスタマイズ可能） |
| **マーカーサイズ** | 10cm × 10cm（デフォルト、設定で変更可能） |
| **検出速度** | リアルタイム（30FPS以上） |
| **複数マーカー** | 対応（最初のマーカーを優先） |
| **照明耐性** | 中程度の照明条件で動作 |

### 3.2 ポーズ推定機能

| 要件 | 詳細 |
|------|------|
| **推定手法** | SolvePnP（Perspective-n-Point） |
| **入力データ** | マーカーコーナー座標（4点） |
| **キャリブレーション** | center.json から読み込み |
| **出力データ** | 回転ベクトル (rvec) + 並進ベクトル (tvec) |
| **座標系** | カメラ座標系 |
| **精度** | キャリブレーションデータに依存 |

### 3.3 3D 表示機能

#### フラスタム（緑）
- カメラの視野領域を四角錐で表現
- 原点からスケーリングされた相対座標で表示

#### ボディ（青）
- カメラの物理的な形状を直方体で表現
- 寸法：幅 × 高さ × 奥行き

#### 光軸矢印（黄）
- カメラの撮影方向（Z軸正方向）を矢印で表示
- 先端にアロー形状

### 3.4 UI 表示機能

**ウィンドウ 1: カメラ映像＋マーカー＋オーバーレイ**
- リアルタイム カメラ映像
- 検出されたマーカーを枠線で表示
- マーカーID をテキスト表示
- 3D ワイヤフレーム（投影）をオーバーレイ
- デバッグ情報（マーカー数、距離、角度）

**ウィンドウ 2: 3D ワイヤフレーム独立ビュー**
- マーカーを中心とした3D座標系
- カメラ ワイヤフレームをスケーリングして表示
- 背景色：暗いグレー

### 3.5 設定機能

## 6. アルゴリズム仕様

### 6.1 マーカー検出アルゴリズム

**処理フロー**:

1. **フレーム入力**: BGR 色空間のカメラフレーム（H×W×3）

2. **グレースケール変換**:
   ```
   gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
   ```

3. **ArUco 検出**:
   ```
   corners, ids, rejected = detector.detectMarkers(gray)
   ```
   - corners: マーカーのコーナー座標 (N, 4, 2)
   - ids: マーカーID (N,)
   - rejected: 検出失敗の候補

4. **マーカー情報抽出**:
   - マーカー中心: $(x_c, y_c) = \frac{1}{4}\sum_{i=0}^{3}(corner_i)$
   - 回転角: $\theta = \arctan2(corner_1.y - corner_0.y, corner_1.x - corner_0.x)$

### 6.2 ポーズ推定アルゴリズム

**3D オブジェクトモデル**:

マーカーをZ=0平面上の正方形として定義：

$$
P_{obj} = \begin{pmatrix}
-s/2 & s/2 & s/2 & -s/2 \\
-s/2 & -s/2 & s/2 & s/2 \\
0 & 0 & 0 & 0
\end{pmatrix}
$$

ここで $s$ はマーカーサイズ（メートル）

**SolvePnP 処理**:

```python
success, rvec, tvec = cv2.solvePnP(
    objectPoints=P_obj,      # 3D オブジェクト座標
    imagePoints=P_img,       # 2D 画像座標
    cameraMatrix=K,          # カメラ行列
    distCoeffs=dist_coeffs,  # 歪み係数
    flags=cv2.SOLVEPNP_ITERATIVE
)
```

**出力**:
- `rvec`: 回転ベクトル (3,) - Rodrigues形式
- `tvec`: 並進ベクトル (3,) - メートル単位

**回転行列への変換**:
$$
R = exp([\rvec]_\times) \text{ (Rodrigues公式)}
$$

### 6.3 座標系定義

**マーカー座標系**:
- 原点: マーカーの中心
- Z軸: 上向き（マーカー平面の法線）
- X軸: マーカーの右方向
- Y軸: マーカーの下方向

**カメラ座標系**:
- 原点: カメラの光学中心
- Z軸: 前方（撮影方向）
- X軸: 右方向
- Y軸: 下方向

**画像座標系**:
- 原点: 画像の左上
- X軸: 右方向
- Y軸: 下方向

## 8. ファイルフォーマット仕様

### 8.1 center.json（キャリブレーションファイル）

```json
{
  "imageSize": [width, height],
  "cameraMatrix": [
    [fx,  0, cx],
    [ 0, fy, cy],
    [ 0,  0,  1]
  ],
  "distCoeffs": [[k1, k2, p1, p2, k3]],
  "rvecs": [...],
  "tvecs": [...]
}
```

**パラメータ意味**:
- `imageSize`: キャリブレーション用画像解像度 (width, height)
- `cameraMatrix`: 3×3 カメラ内部パラメータ行列
  - `fx, fy`: 焦点距離 (ピクセル単位)
  - `cx, cy`: 画像中心座標 (ピクセル単位)
- `distCoeffs`: 5 個のレンズ歪み係数
  - `k1, k2`: 放射ひずみ係数
  - `p1, p2`: 接線ひずみ係数
  - `k3`: 高次放射ひずみ係数
- `rvecs`: 回転ベクトル (キャリブレーション用)
- `tvecs`: 並進ベクトル (キャリブレーション用)

## 9. エラーハンドリング

### 9.1 エラー分類

| エラー | 原因 | 対応 |
|-------|------|------|
| **カメラ接続エラー** | USB接続なし | 初期化失敗で終了 |
| **キャリブレーション読込失敗** | ファイル形式エラー | 初期化失敗で終了 |
| **マーカー未検出** | マーカーが視野外/暗い | 処理継続（ポーズなし） |
| **ポーズ推定失敗** | 不正なマーカー | 処理継続（ポーズなし） |
| **ウィンドウ表示エラー** | ディスプレイエラー | ログ出力して継続 |

### 9.2 リカバリー戦略

- **軽微エラー**: ログ出力して処理継続
- **重大エラー**: エラーメッセージ表示して正常終了

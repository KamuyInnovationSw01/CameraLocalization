# 💡 開発レッスンと改善パターン (Camera Localization)

## 📌 1. 3Dワイヤーフレームウィンドウが真っ黒になる問題

### 🔴 遭遇した問題
ビデオウィンドウ（カメラ映像）でマーカーが正しく検出されているにもかかわらず、もうひとつの Wireframe ウィンドウに何も描画されず文字「3D View: Camera Pose + Marker Axes...」だけが表示され、それ以外は真っ黒（暗いグレー）になる。

### 🔍 根本原因
1. **設定フラグのデフォルト無効化**:
   `config.yaml` で `enable_3d_render: false` となっており、3D表示部が完全にバイパスされていた。
2. **ピクセル投影座標の二重スケーリングバグ**:
   `wireframe_renderer.py` の従来の `draw_3d_view` メソッド内で、一連 of 3D 頂点を `project_3d_to_2d` を用いて既に画像ピクセル座標（例：x=480, y=360）に投影したのに対し、さらに `scale_factor = 150` を乗算し、画面中心のピクセルオフセットを加算していた。
   これにより、投影されたすべての頂点が `x = 480 * 150 + 480 = 72480` のようになり、ウィンドウ解像度（960x720）をはるかに超えて宇宙の彼方に吹き飛んでいた。その結果、画面内に線や円が一切描画されなくなっていた（直値で描画されるテキストのみ表示されていた）。

### ✨ 解決策（エレガントなパターン）
- **仮想対話カメラ (LookAt) による正規3D投影の実装**:
  ピクセル座標に対する誤った二重変換を排し、マーカー（世界座標の原点）とカメラ（世界座標系に逆走投影したポーズ）を、3次元空間のまま別角度から捉える「仮想第3者カメラ（LookAt）」処理を構築。
  仮想カメラアングルから一定の焦点距離 `K_virtual` を用いて `cv2.projectPoints` を行うことで、あらゆるポーズでもマーカー（赤）とカメラの形状（オレンジ・シアン）が、完全にピクセル画面中央にアスペクト比を狂わせずに自動フィットして描画されるように大刷新した。
- **安全なエラーフォールバック＆軽量化**:
  CPU負荷が極めて高い matplotlib版 の代わりに、この高速な OpenCV版 `draw_3d_view` をデフォルトに設定。エラー時には相互にフォールバック可能とした。

### 📖 学び（レッスン）
- **投影座標の物理的意味の意識**:
  `cv2.projectPoints` やカメラ行列 `K` を適用した時点で、その座標系は「ミリメートル・メートル単位の実世界座標」から「ピクセル単位の画像座標」へと完全に変換されている。そこにスケーリングや並進を再度適用すると表示が崩壊するため、変形は必ず投影前の **実世界空間（3D）段階** で行うべきである。

## 📌 2. 姿勢後処理による前後位置の異常

### 根本原因
`solvePnP` が返す `rvec` と `tvec` は同じ剛体変換を構成するため、`tvec` だけを反転してはいけない。また、`R @ diag([1, 1, -1])` は行列式が `-1` の鏡映変換であり、回転行列として `Rodrigues` に渡すと再投影姿勢が壊れる。

### 対応
`pose_estimator.py` では `solvePnP` の `rvec` と `tvec` をそのまま返し、カメラ位置だけを `-R.T @ tvec` で計算する。座標系の見せ方は `wireframe_renderer.py` の表示側で行う。

## 📌 3. 右手系3Dビューと自動フィット

- OpenCVの投影に合わせて、仮想LookAt行列の行列式を `+1` に保つ。
- マーカーの `+Z`（正面方向）が画面上向きになるよう、LookAtの左右軸を構築する。
- マーカー、座標軸、カメラ本体、フラスタム、光軸の全頂点から投影範囲を求め、焦点距離と主点を自動調整する。
- 画面端から50pxの余白を確保し、カメラが3Dビュー外へ出ないようにする。

## 📌 4. 再投影誤差の記録

姿勢推定直後に4隅を再投影し、コンソールへ平均・最大誤差を `pix` 単位で出力する。生の姿勢が画像を説明できているかを、補正後の見た目だけに頼らず確認できる。
